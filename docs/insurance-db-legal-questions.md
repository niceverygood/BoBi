**의뢰인:** 주식회사 바틀 (Bottle Inc.) — 한승수 대표
**작성일:** 2026-05-06
**관련 문서:** [insurance-db-research.md](./insurance-db-research.md) (Phase 1 사전 조사 보고서, 528줄)

> 본 문서는 변호사 사무소에 그대로 송부 가능하도록 작성됨. 사실관계는 보고서 §1~§6,
> 자문 항목은 §4.2 + §7.0의 (D) 하이브리드 트랙 채택 시 발생하는 추가 항목 기준.

## 1. 의뢰 배경

당사는 한국 보험설계사 1,200+명이 사용하는 SaaS "보비"를 운영 중이며, 사용자에게
**가입 가능 보험상품을 매칭·비교**하기 위한 보험상품 DB를 구축하려 합니다.

데이터 출처는 (1) 정부·협회 공시, (2) 개별 보험사 공식 사이트, (3) 향후 마이데이터
사업자 제휴 — 세 갈래입니다.

본 자문에서 결정해야 하는 것은 다음 4건입니다 (각 항목별 구체적 질문은 §3).

| # | 항목 | 우선순위 |
|---|---|---|
| 1 | 보험사·협회 공시 데이터 자동 수집(크롤링)의 합법성 | 🔴 필수 |
| 2 | 수집한 공시 데이터의 2차 가공·재배포 시 신고 의무 | 🔴 필수 |
| 3 | 약관 PDF의 저작권·인용 한도 | 🔴 필수 |
| 4 | 인수기준(가입 가부) 데이터 수집의 영업비밀·부정경쟁방지법 위반 가능성 | 🔴 필수 |
| 5 | 마이데이터 사업자(뱅크샐러드/카카오페이) 제휴 계약 시 검토할 표준 조항 | 🟡 트랙 (D) 채택 시 |

## 2. 사실관계 요약

### 2.1 수집 대상 사이트

| 사이트 | 운영 주체 | 데이터 성격 | robots.txt 정책 |
|---|---|---|---|
| 보험다모아 (`e-insmarket.or.kr`) | 생보협회 + 손보협회 공동 | 통합공시 (보험료·보장 비교) | 직접 fetch 시 403 (WAF 차단) |
| 손보협회 공시 (`kpub.knia.or.kr`) | 손해보험협회 | 공시 의무 데이터 | 직접 fetch 시 403 |
| **생보협회 공시 (`pub.insure.or.kr`)** | 생명보험협회 | 공시 의무 데이터 | **`User-agent: * / Disallow: /` — 명시 전체 차단** |
| 금감원 비교공시 (`fine.fss.or.kr`) | 금융감독원 | 감독기관 공시 | 일부 path만 disallow, 그 외 허용 |
| 손보 소비자포털 (`consumer.knia.or.kr`) | 손해보험협회 | 공시·민원·약관 해설 | robots.txt 미존재 |
| 개별 보험사 10개사 | 각 회사 | 자사 상품·약관 PDF | 5개사 전면 허용, 1개사 사실상 차단(DB손보), 1개사 Naver봇만 명시 허용(KB손보), 3개사 응답 불안정 |

### 2.2 수집 데이터 종류

1. 상품 마스터 (회사·상품명·판매상태·출시일)
2. 약관 (PDF 원본 + OCR 텍스트)
3. 보장내역 (담보명·가입금액 한도)
4. 보험료 요율 (연령·성별·흡연 조건별 월보험료)
5. **인수기준** (질병 코드별 가입 가능 여부 — 가장 민감한 항목)

### 2.3 수집 후 활용 방식

- 자사 SaaS "보비" 내에서 보험설계사가 고객 위험도 분석 결과와 매칭하여 **가입 가능
  상품 후보**를 보여주는 기능에 활용
- 사용자에게 노출 시 **출처·수집일·공시 의무 데이터 여부**를 함께 표기 예정
- 외부 재배포·API 판매 계획 없음 (현재 단계)

### 2.4 회사 자본 상태

- 마이데이터 인가 자본금 요건(5억원)을 단기 충족하기는 어려운 단계
- 따라서 마이데이터 인가 직접 취득(트랙 B) 대신 **하이브리드(트랙 D — 안전한 출처
  직접 크롤링 + 마이데이터 사업자 제휴 병행)** 를 검토 중

## 3. 자문 항목

### 3.1 [필수] 보험사·협회 공시 데이터 자동 수집의 합법성

#### 질문

1. **robots.txt에서 일반 봇을 명시 차단(Disallow: /)** 한 사이트(생보협회 공시
   `pub.insure.or.kr`)를 자동화된 도구로 수집할 경우, 다음 각 법령에서 위반
   가능성이 있는지:
   - 정보통신망법 제48조 (정보통신망 침해)
   - 부정경쟁방지법 제2조 1호 카목 (성과 도용)
   - 데이터베이스 보호 (저작권법 제93조)
   - 형법상 업무방해
2. **403 응답으로 차단된 사이트**(보험다모아·손보협회·소비자포털)에 대해 일반 브라우저
   User-Agent로 우회하여 접근하는 행위가 ToS 위반·전자기록 변경 등에 해당하는지
3. robots.txt 미명시 + 응답 정상인 사이트(금감원 등)는 **공시 의무 데이터인 만큼
   자동 수집이 적법하다고 볼 수 있는지** — 판례·금감원 유권해석 사례

#### 우리 검토 의견 (변호사 검증 부탁)

- 생보협회 공시 직접 크롤링은 ❌ — robots 명시 차단 + ToS 별도 확인 시 위반
  가능성 큼. **수집 대상에서 제외** 권장.
- 금감원 공시는 공익적 성격이 강하고 robots도 일부 path만 차단하므로 ✅ 가능.
- 협회 사이트는 협조 요청 메일 또는 PDF 일괄 다운로드(수동) 우회 권장.

---

### 3.2 [필수] 공시 데이터 2차 가공·재배포 시 신고 의무

#### 질문

1. 보험상품 공시 데이터를 수집·정규화해서 자사 사용자(보험설계사)에게 **검색·비교
   기능**으로 제공할 경우, 다음 신고·등록 의무가 발생하는지:
   - 보험업법상 **보험상품 비교·공시 사업자** 등록 (보험업법 제124조의2 등)
   - 금융소비자 보호에 관한 법률상 등록 (금소법 제12조)
   - 정보통신망법 부가통신사업자 신고 외 별도 의무
2. "비교" 표시 자체가 보험중개·모집 행위로 의제될 위험은 없는지 (현재 보비는
   설계사용 도구이며 일반 소비자에게 직접 판매하지 않음 — 이 차이가 면책 사유가 되는지)
3. 데이터 정확성 책임은 누구에게 귀속되는지 (수집한 공시 데이터에 오류가 있을 때
   당사·원 출처·플랫폼 중)

#### 우리 검토 의견 (변호사 검증 부탁)

- B2B 도구(설계사용)이고 일반소비자 노출이 아니므로 보험업법 124조의2 적용 외일
  가능성 — 다만 "사용자(설계사)가 자기 고객에게 보비 화면을 그대로 보여주는 경우"의
  법적 위치 명확화 필요.

---

### 3.3 [필수] 약관 PDF의 저작권·인용 한도

#### 질문

1. 보험약관 PDF는 보험업법상 공시 의무 사항인데, **PDF 자체의 저작권**은 보험사에
   귀속되는지 / 공중에 제공된 자료로 자유 인용 가능한지
2. 자사가 약관 PDF를 (1) 수집·저장(Cloudflare R2)하고 (2) 텍스트 추출(OCR)해서
   (3) 사용자에게 검색 가능한 형태로 제공할 때, 각 단계에서 저작권 침해 또는
   계약 위반 가능성
3. **사용자에게 노출 시 원본 PDF 링크를 함께 제공**하면 인용 적법성에 차이가 있는지

#### 우리 검토 의견 (변호사 검증 부탁)

- 공시 의무 자료로서 공중송신권 적용 제한 가능 — 저작권법 제35조의5 "공정이용"
  요건 검토 필요.
- 원본 링크 동시 제공은 인용 적법성을 강화하는 요소.

---

### 3.4 [필수] 인수기준 데이터 수집의 영업비밀·부정경쟁방지법

#### 질문

1. 보험사 "인수기준"(질병별 가입 가부, 할증·부담보 조건)은 보험사 영업비밀로
   분류되는 자료입니다. 다음 수집 경로의 합법성:
   - 보험사 직접 협조 요청 → ✅ 추정
   - 보험사 공식 공시 (있는 경우) → ✅ 추정
   - **GA 영업소·설계사 사이에 공유되는 인수가이드** → ❌ 영업비밀 침해 추정,
     명확한 자문 필요
   - 사용자(설계사)가 보비에 직접 입력 → 사용자 책임? 플랫폼 공동 책임?
2. 사용자가 인수가이드를 보비에 업로드해서 다른 사용자와 **공유**하는 기능을
   제공할 경우, 부정경쟁방지법 제2조 1호 차목·카목 위반 가능성

#### 우리 검토 의견 (변호사 검증 부탁)

- 인수기준은 **보험사 정식 협조 채널 외 경로로는 절대 수집 금지** — 영업비밀
  침해 위험 큼.
- 사용자 업로드·공유 기능은 보류 (Phase 3 이후 검토).

---

### 3.5 [트랙 D 채택 시] 마이데이터 사업자 제휴 계약 표준 조항

#### 질문

당사가 뱅크샐러드·카카오페이 등 본인신용정보관리회사(마이데이터 사업자)와
제휴해서 보험 데이터를 받기로 협상 시, 표준 계약서에서 검토해야 할 핵심
조항 5가지를 정리해주십시오:

1. **데이터 사용 범위** — 보비 SaaS 내 사용자 매칭에만 한정 vs 자사 데이터로 추가
   가공·학습 가능 여부
2. **재배포 권한** — API/리포트 형태로 보비 사용자(설계사)에게 노출 시 제휴사
   동의 필요 여부, 출처 표기 의무
3. **사용자 동의 흐름** — 보비 사용자(설계사)가 자기 고객 데이터를 매칭에
   활용할 때, 본인동의는 누가 받는지 (보비? 제휴사? 둘 다?)
4. **데이터 정확성·면책** — 제휴사 데이터에 오류가 있을 때 책임 분담
5. **계약 종료 시 데이터 처리** — 제휴 종료 후 기존 데이터 보존·삭제·이관 방식

## 4. 부탁드릴 산출물

자문 결과로 받기를 희망하는 형식:

1. 각 항목별 (3.1~3.5) **결론(가능/조건부 가능/불가)** + **근거 법령·판례** + **권장 사항**
2. **변호사 의견서** 형식 (필요 시 정식 의견서 발급도 가능한지 별도 안내 부탁)
3. 자문료 견적 + 예상 소요 시간
