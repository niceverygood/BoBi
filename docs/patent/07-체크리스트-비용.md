## 🗓 전체 로드맵 (권장 순서)

```
[D-0]       최초 공개일 확정 (공지예외 주장 기준일 산정)
  ↓
[D+1~7]    선행기술조사 직접 수행 (키프리스 + 구글 특허)
  ↓

[D+180~365] 보정서 제출 → 등록결정 → 특허료 납부 → 특허증 발급
```

| 대상 | 예상 분류 | 비용 (감면 후) |
|---|---|---|
| "보비" 문자 | 9류 (SW) + 36류 (보험/금융) + 42류 (SaaS) | **약 20~30만원/건** |

| 로고(심볼) | 위 전부 | 동일 |

→ 상표는 사용 증빙과 독점권이 명확해 **스타트업 필수 자산**. 특허보다 먼저 진행 권장.

### 2. 저작권 등록 (선택)

- 소스코드, UI/UX 디자인, 명세서, 프롬프트 등
- 한국저작권위원회 등록, 건당 약 7만원
- 권리분쟁 시 입증 용이 (등록 = 법적 추정 효과)

### 3. 영업비밀 관리

- 프롬프트 원문, 임계치 수치(DECLINE_RISK_CODES 가중치 등), AI 모델 파인튜닝 데이터
- **영업비밀로 보호 = 공개하지 않음**. 특허는 공개되므로 "특허로 공개할 것 vs 비밀로 유지할 것" 전략적 분리 필요.
- 권장: 알고리즘 개념은 특허로, 실제 가중치 테이블은 영업비밀로 유지 (명세서에는 일반적 서술)

### 4. 계약·규정 정비

- 직무발명 규정
- 개발자 입사 계약서 내 특허권 귀속 조항
- NDA (외주 개발자/파트너)
- 데이터 처리 위수탁 계약 (의료정보 처리 관련)

## 📞 출원 전 최종 확인 Q&A

**Q1. 이미 서비스 중인데 특허 가능한가?**
- A. 최초 공개일로부터 **12개월 이내**이고 **공지예외주장**을 정확히 수행하면 가능.

**Q2. 여러 기능을 한 건으로 묶을지, 분리할지?**
- A. 단일성 원칙상 "하나의 발명 개념"에 묶어야 함. 본 발명은 **"미래의 나 파이프라인"** 이라는 단일 개념으로 묶임. 후속으로 **분할출원** 또는 **별도 출원** 으로 확장 가능.

**Q3. LLM 호출 자체도 특허 가능?**
- A. 단순 LLM 호출은 특허 대상 아님. 본 발명은 **"특정 프롬프트 구조 + 3시나리오 산출 로직 + 위험 점수 휴리스틱의 결합"** 이므로 진보성 있음.

**Q4. 해외 특허도 필요한가?**
- A. 현재 서비스가 한국 시장 중심이면 한국 등록 후 PCT 국제출원(30개월 내)으로 확장 가능. 초기엔 한국 1건에 집중 권장.

**Q5. 심사에서 거절되면?**
- A. 의견서/보정서로 대응 가능. 1차 거절 → 2차 거절 → 거절결정 → 특허심판원 심판청구 → 특허법원 소송까지 단계적. 대부분 1~2차 보정에서 해결됨.

**Q6. 이미 타사가 특허 가진 기능이 있으면?**
- A. 본 발명의 **차이점**을 청구항에 명확히 표현. 심사관이 차이를 인정하면 등록 가능. 선행기술 조사(06)가 중요한 이유.

**Q7. 특허를 상대방이 무효화할 수 있나?**
- A. 가능. 특허등록 후에도 누구나 무효심판 청구 가능. 따라서 **출원 단계의 충실한 청구항 + 명세서 품질**이 중요.
