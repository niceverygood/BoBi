## 🎯 출원 전략 요약

### 메인 출원 (1건, 우선)
**발명의 명칭**: "AI 기반 보험 미래 시나리오 통합 비교 방법 및 시스템"
- **핵심**: 진료이력 분석 → 질병위험도 예측 → 질병별 비용 시뮬레이션 → 5년 후 인수 가능성 추정 → 3가지 시나리오(A 지금 보완 / B 5년 후 가입 / C 미조치) 통합 비교 출력을 **하나의 오케스트레이션**으로 수행하는 방법

### 추가 분할 출원 후보 (예산 여유 시)
1. **5년 후 보험 인수 가능성 추정 휴리스틱**
   - 현재 진단코드 가중치 + 위험도 리포트 상위 질환 × 카테고리 가중치 + 다약제 가산 + risk flag 점수 → 0~100 정규화 → 3단계 outcome + 보험료 멀티플라이어 + 면책 예측
2. **AI 기반 보험 고지사항 자동 분류 + 서버사이드 날짜 재검증**
   - PDF/CODEF API → LLM 1회 호출로 3개월/1년/5년/상시복용약 카테고리 자동 분류 → LLM 기간 판정을 서버가 오늘 날짜 기준으로 재검증/보정

### 권리 보호 방식 비교

| 보호수단 | 대상 | 비용 | 권장도 |
|---|---|---|---|
| **특허** (발명특허) | 오케스트레이션 방법, 휴리스틱 알고리즘 | ~5백만원/건 | ⭐⭐⭐ |
| **상표** | "보비", "BoBi", 로고, "미래의 나" | ~30만원/류 | ⭐⭐⭐⭐ (필수) |
| **저작권** | 소스코드, UI 디자인 | 무료(자동), 등록 7만원 | ⭐⭐ |
| **영업비밀** | 프롬프트, 룰엔진 임계값 | 비밀관리체계만 갖추면 자동 | ⭐⭐⭐ |

## ⚠️ 필독 사항

### 1. 신규성 상실 방어 (공지예외주장)
- 보비가 이미 서비스 중이라면 **최초 공개일로부터 12개월 이내 출원 필수**
- 출원서 제출 시 "공지예외적용주장" 체크 + 최초 공개 증빙(블로그 URL, 앱스토어 출시일, 보도자료 등) 첨부
- **최초 공개일을 확정**하세요: git log, 앱스토어 출시일, 랜딩페이지 배포일, SNS 게시일 중 가장 앞선 날짜

### 2. 컴퓨터 관련 발명 특유 요건
- 특허법 제29조 + 심사기준 "컴퓨터 관련 발명 심사기준" 준수
- **단순 영업방법(BM)**만으로는 거절 → **"컴퓨터에 의해 구현되는 구체적 기술적 수단"** 강조 필요
- 해결수단에 서버/DB/AI 모델/API 호출 구조를 명시해야 기술성 인정

### 3. 진보성 인정 포인트
다음을 명세서에서 강조:
- **통합성**: 위험도 + 비용 + 인수추정을 하나의 파이프라인으로 결합 (기존은 각각 별개 서비스)
- **개인화**: 환자별 진료코드 기반 휴리스틱 (일반 통계가 아닌 개인별 점수)
- **시간축**: "5년 후 미래 시점" 시뮬레이션을 현재 데이터로 추정하는 점
- **3시나리오 대비**: 설계사-고객 상담 맥락에서 의사결정 지원 구조
